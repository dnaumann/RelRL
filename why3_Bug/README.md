# Why3 Translation Bug

WhyRel translates `prog.rl` to `prog.mlw`. The WhyRel expression:

`([> True |> -> [> True |>) /\  ([> True |> -> [> True |>)` is translated to

`(true -> true) /\ true -> true`. The precedence is incorrectly translated.

- The AST of the WhyRel expression is correctly generated.
- The Ptree module creates the right why3 AST. 
  - See sexp.txt. (generated by inserting why3 print instruction in main.ml:81)
- The bug appears when writing to mlw file using Mlw_printer module.
